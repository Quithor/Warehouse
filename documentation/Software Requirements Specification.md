# 一、引言

## 1. 项目背景和目标

各大超市百货的商品种类繁多，单靠人力管理成本极高。

本项目就是诞生在这样的环境下，提供一种软件上的解决方案。

面向超市以及各大有需要的物流仓库，提供一种软件上的管理方案。

## 2. 项目范围

项目遵循MIT协议进行开源

# 二、 项目概述

## 1. 软件目的

体验完整的软件生命周期，培养团队协作能力，加深编程语言及框架的熟悉

## 2. 功能详细描述

1. 登录
   - 输入 用户输入账号
   - 输出 显示是否登录失败，登录成功自动跳转到用户的工作主页
   - 业务规则
     服务端向客户端发送post请求，客户端接收post请求去访问数据库查询用户是否存在，如果存在再判断用户输入 密码是否正确，并判断用户身份，当用户密码正确登录成功后将所有信息返回服务器
2. 注册
   - 输入 输入用户的信息
   - 输出 用户命是否被占用、密码的安全性、注册是否成功等
   - 业务规则
   浏览器将输入信息打包以一个post请求发送到服务器，服务器对数据进行检查：用户的账号是邮箱或中国大陆手机号(+86)、账号是否已被注册
   在所有检查都通过时向浏览器返回注册成功的信息

### 2.1 s端

1. 销售
   - 输入 消费者购买的商品列表
   - 输出 发票、商品清单
   - 业务规则  
   浏览器扫码机模拟用户选购的商品
2. 库存申请
   - 输入 商品类型
   - 输出 申请发送成功，等待审核
   - 业务规则  
   售货员点击货架告警时像数据库写入一条记录
3. 营收统计
    - 输入 统计时间
    - 输出 当天营收
    - 业务规则
    售货员点击统计并输入统计的日期，后端计算当天营收并发送给前端进行显示

### 2.2 r端

1. 入库
    - 输入 相关商品信息
    - 输出 该商品当前库存
    - 业务规则  
    前端将商品信息传到后端，在后端使用JDBC将数据添加到数据库，并访问该商品当前的数量，添加入库记录
2. 出库
    - 输入 出库商品的信息以及商品数量
    - 输出 当前库存
    - 业务规则  
    从库存列表中点击商品的出库按钮对商品进行出库，后端对前端发出的信息进行处理
3. 处理申请
    - 输入 处理结果
    - 输出 （None）
    - 业务规则  
    在仓管登录到系统时弹出所有申请，多页的方式展示，当前处理完成后跳转到下一条，处理结果发送到后端进一步处理
4. 监控
    - 输入
    - 输出
    - 业务规则  
    前端每分钟发送数据刷新的申请像后端请求数据，后端接收后反馈数据给前端

# 七、系统约束

## 1. 技术约束

### 1.1 开发环境
本系统需要在支持Java 11或更高版本的集成开发环境（IDE）中进行开发。  
使用的数据库系统应为关系型数据库管理系统（RDBMS）MySQL 8.0.33。  
前端开发应兼容现代浏览器，包括但不限于Microsoft Edge、Chrome、Firefox、Safari的最新版本。    

### 1.2 编程语言与框架
主要编程语言为Java，使用Spring Boot框架进行后端开发。  
前端开发使用vite框架。  
前后端通信采用JSON格式的数据交换。  

### 1.3 第三方库与工具
允许使用经过充分测试且广泛使用的第三方库和工具，fastjson2、mysql-connection-java。  

## 2. 法律和政策约束

### 2.1 数据隐私与保护
用户有权访问、更正和删除其个人数据。  

### 2.2 知识产权
系统不得侵犯任何第三方的知识产权，包括但不限于版权、商标和专利。  
所有使用的开源代码和库必须遵循其许可协议。  

### 2.3 法律法规遵守
本系统开发遵守中华人民共和国法律。  
系统不得用于任何非法活动或违反道德的行为。  

## 3. 其他约束要求
本系统尽量提供用户友好的界面和清晰的错误提示，以提高用户体验。  
系统支持多语言，以满足不同用户的需求。用户可以根据本系统的百科编写语言包使用，系统默认包含简体中文(Simple Chinese)和English(United States)两种语言。`
