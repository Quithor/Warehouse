# 一、引言

## 1. 项目背景和目标

各大超市百货的商品种类繁多，单靠人力管理成本极高。

本项目就是诞生在这样的环境下，提供一种软件上的解决方案。

面向超市以及各大有需要的物流仓库，提供一种软件上的管理方案。

## 2. 项目范围

项目遵循MIT协议进行开源

## 3. 相关文档和参考

- [用户需求说明书](User%20Requirements%20Specification.md)
- [Spring boot官方文档](https://spring.io/projects/spring-boot)

# 二、 项目概述

## 1. 软件目的

体验完整的软件生命周期，培养团队协作能力，加深编程语言及框架的熟悉

## 2. 主要功能

库存管理，商品售出统计，商品出售

## 3. 用户群体

超市内的售货员及仓管

## 1.功能模块划分

|  功能  |        模块        |
|:----:|:----------------:|

|  出库  |      output      |

|  监控  |       view       |

## 2. 功能详细描述

1. 登录
   - 输入 用户输入账号
   - 输出 显示是否登录失败，登录成功自动跳转到用户的工作主页
   - 业务规则
     服务端向客户端发送post请求，客户端接收post请求去访问数据库查询用户是否存在，如果存在再判断用户输入 密码是否正确，并判断用户身份，当用户密码正确登录成功后将所有信息返回服务器
2. 注册
   - 输入 输入用户的信息
   - 输出 用户命是否被占用、密码的安全性、注册是否成功等
   - 业务规则
   浏览器将输入信息打包以一个post请求发送到服务器，服务器对数据进行检查：用户的账号是邮箱或中国大陆手机号(+86)、账号是否已被注册
   在所有检查都通过时向浏览器返回注册成功的信息

### 2.1 s端

1. 销售
   - 输入 消费者购买的商品列表
   - 输出 发票、商品清单
   - 业务规则  
   浏览器扫码机模拟用户选购的商品
2. 库存申请
   - 输入 商品类型
   - 输出 申请发送成功，等待审核
   - 业务规则  
   售货员点击货架告警时像数据库写入一条记录
3. 营收统计

### 2.2 r端

1. 入库
2. 出库
3. 处理申请
4. 监控
